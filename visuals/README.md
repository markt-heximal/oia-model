# /visuals — Architectural diagrams

The visual reference for the model. The three canonical diagrams are now available as **SVG**
(crisp, web-friendly, editable — the source of truth for the diagrams) alongside the original **PDF**
renders.

## Contents

- **`OIA-Stack-v0.5.svg`** / **`.pdf`** — the ten-layer stack diagram (Layer 0 substrate, Layers 1–9 the
  stack) with the seven cross-layer spans.
- **`OIA-Threat-Surfaces-v0.5.svg`** / **`.pdf`** — the Four Named Threat Surfaces diagram (Mimicry,
  Agent-in-the-middle, Agent spoofing, Model compromise) and their concentration in the operational stack.
- **`OIA-Lens-Overlay-v0.5.svg`** / **`.pdf`** — the Lens Overlay showing how the three audience lenses map
  onto the architecture (and the Board lens's spans-first inversion).
- **`OIA-Architecture-with-Framework-References-v0.5.pdf`** — the architecture annotated with the external
  frameworks it composes with (OSI, NIST, MITRE, OWASP, CCC, ISO/IEC 42001).
- **`OIA-Visual-Reference-v0.5.pdf`** / **`.docx`** — the diagram set with explanatory text.

## The three canonical diagrams

Per the Body of Work Framing, the Visual Reference comprises three diagrams: **the Stack**, **the Lens
Overlay**, and **the Four Named Threat Surfaces**. The `.svg` files are newly authored, reproducible
renditions built to the [design system](../docs/DESIGN-SYSTEM.md) (generated from a script for precise
geometry); the `.pdf` files are the original renders. The two are visually distinct but describe the same
canonical content.
