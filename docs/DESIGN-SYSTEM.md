# OIA Design System

The visual and editorial conventions for the OIA body of work. The goal is a calm, institutional,
vendor-neutral look — the OIA is a reference model, and its materials should read like a standard,
not a pitch. This document governs the diagrams, the generated documents, and new contributions.

## Palette

| Role | Hex | Use |
|------|-----|-----|
| Navy (primary) | `#14213d` | Headers, titles, layer numbers, primary text emphasis |
| Slate | `#33415c` | Secondary text, captions, descriptors |
| Ink | `#1a1a1a` | Body text |
| Line | `#c9ced6` | Rules, borders, dividers |
| Panel | `#f4f6f9` | Legend/side-panel fills, blockquotes |
| Paper | `#ffffff` | Backgrounds |

**Layer bands** use a single-hue blue ramp from light at Layer 9 (`#e8edf5`) to deep at Layer 0
(`#8c9fc0`), so the stack reads as one system with Layer 0 visually weighted as the substrate.

**Accent colors** are reserved and meaningful — never decorative:

| Meaning | Hex |
|---------|-----|
| Mimicry / threat / emphasis | `#c8102e` (red) |
| Sovereignty / technology lens | `#1f7a4d` (green) |
| Agent-in-the-middle / auditability | `#b8860b` (gold) |
| Identity / agent spoofing / board lens | `#2b5fb3` (blue) |
| Provenance / model compromise | `#7a3fa0` (violet) |
| Verifiability | `#0f766e` (teal) |
| Energy & environmental | `#5a5a5a` (grey) |

## Typography

- **Headings & UI:** a neutral sans — Helvetica Neue / Arial / Liberation Sans.
- **Body (documents):** a readable serif — Liberation Serif / Georgia — at ~11.5pt, 1.42 line height.
- **Diagram labels:** sans throughout.
- Titles navy, bold; section headings navy with a thin `#c9ced6` underline; captions slate.

## Diagram conventions

- **Bottom-up stack.** Layer 9 at top, Layer 0 at bottom; Layer 0 separated by a gap and labeled
  "substrate"; Layers 1–9 bracketed as "the architectural stack."
- **Layers** are rounded rectangles (`rx≈7`) with a circular navy number badge, bold navy name, and a
  one-line slate descriptor.
- **Spans** are rendered as a labeled side panel with colored dots (spans thread *vertically*; they are
  not layers). Verifiability is tagged "(new)" in red.
- **Threats** are drawn as colored paths/markers over the stack, with a legend — origin and propagation
  shown (e.g. Mimicry L9→L7→L3).
- **Lenses** are colored cards connected to the layers/spans they emphasize; the Board lens connects to
  the spans rail to signal its spans-first inversion.
- Every diagram carries the header band (title + subtitle + "AGENTICS FOUNDATION · OIA v0.5") and a
  one-line footer of context.

## Formats & toolchain

- **Diagrams:** authored as **SVG** (source of truth), generated by `scripts`-style Python for precise,
  reproducible geometry. PDF renders may accompany them.
- **Authored prose (lenses, KB, docs):** **Markdown** is the source of truth. DOCX and PDF for the
  document family are generated via `python-markdown` → styled HTML → LibreOffice, using this palette
  and type system.
- **Source documents** (Reference Paper, etc.) are versioned as PDF/DOCX deliverables.

## Editorial voice

- Institutional third person; vendor-neutral; no founder commercial branding in the model body
  (see [ADR-0015](adr/0015-institutional-voice-exclude-founder-branding.md)).
- Use the canonical vocabulary from the [Glossary](GLOSSARY.md) and the Architecture Lock verbatim or
  in faithful paraphrase.
- Every element should be defensible to a skeptical practitioner — a proposition, not a slogan.
