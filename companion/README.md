# /companion — Framing, lock, and historical materials

Supporting and historical artifacts: how the body of work is framed, the canonical spec that keeps it
coherent, and the working materials that produced it.

## Framing & canonical spec

- **`OIA-Body-of-Work-Framing-v0.5.docx`** / **`.pdf`** — why OIA is a *body of work* (five v0.5 artifacts)
  rather than a single document, with a reading guide by role. See [ADR-0014](../docs/adr/0014-publish-as-body-of-work.md).
- **`OIA-v0.5-Architecture-Lock.pdf`** / **`.docx`** — the **canonical internal specification** of what
  changed v0.1 → v0.5 and the vocabulary all family documents must use. The reference against which all
  architectural language is checked. See [ADR-0016](../docs/adr/0016-architecture-lock-as-anti-drift-spec.md).

## Historical / working materials (v0.1 era)

- **`OIA-Model-v0.1-Decision-Log-Digest.docx`** — condensed decision log.
- **`OIA-Model-v0.1-Working-Session-Deck.pptx`** — the working-session deck behind the v0.1 model.
- **`OIA-Model-Presentation-early.pptx`** — an early presentation of the model.
- **`OIA-Model-agenticsfdn-deck.pdf`** — an Agentics Foundation deck of the model.
- **`ruvnet-analysis.pdf`** — *"Toward an AI-Native Operating System Stack"* (independent research note,
  Mar 2026) analysing the ruvnet ecosystem (Ruflo, RuVector, RuView, agentic-flow) as an AI-native OS,
  with a formal interface architecture and P/M/O/C threat model. Architecturally parallel to OIA; not an
  OIA document.

## Excluded from this repository

Some materials that were co-located with the OIA working archive are intentionally **not** included
here, as they are confidential third-party materials unrelated to the vendor-neutral OIA model: the
Nutanix Enterprise AI (NAI) customer overview and Letter-of-Expectations documents, and the
Proofpoint-specific v3.1 strategy maps.
