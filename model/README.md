# /model — The canonical model

The authoritative statement of the OIA Reference Model.

## Contents

- **`OIA-Canonical-Knowledge-Base-v0.5.md`** — the machine-readable single-source-of-truth: the full
  architecture in structured Markdown (ten layers with per-layer detail, seven spans, threats, actors,
  connections, framework mappings, version note). Built for RAG and for downstream artifacts to derive from.
- **`OIA-Reference-Paper-v0.5.pdf`** / **`.docx`** — the ~40-page authoritative reference (v0.5). The
  primary human-facing artifact.
- **`OIA-Model-v0.1-Canonical-KB.docx`** — the original ~23k-word canonical knowledge base (v0.1).
- **`OIA-Model-v0.1-Digest.docx`** — the Reader's Digest condensation (v0.1).
- **`OIA-Model-v0.1-Decision-Log.docx`** — the record of naming and framing decisions (see also the
  repo-native [ADRs](../docs/adr/)).

## Reading order

1. `OIA-Reference-Paper-v0.5.pdf` — the current architecture in full.
2. `OIA-Model-v0.1-Digest.docx` — a faster pass if you want the shape before the depth.
3. `OIA-Model-v0.1-Decision-Log.docx` — why the model is shaped as it is.

## Note

`OIA-Canonical-Knowledge-Base-v0.5.md` is the Markdown single-source-of-truth (authored 2026-07-01 from
the Reference Paper, the v0.1 model, and the Architecture Lock). It is intended as the source other
artifacts derive from; the author should review it before it is treated as canonical.
